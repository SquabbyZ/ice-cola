# 扩展商店功能测试指南

## 📋 测试概述

本文档提供了扩展商店所有功能的详细测试步骤和验证点。所有测试必须使用**真实数据**，禁止使用 mock。

---

## ✅ 测试前准备

### 1. 环境检查
```bash
# 确认前端服务运行在 1420 端口
curl http://localhost:1420

# 确认后端服务运行（默认 3000 端口）
curl http://localhost:3000/api/extensions

# 确认数据库中有扩展数据
docker exec -it ice-cola-server psql -U postgres -d openclaw -c "SELECT COUNT(*) FROM extensions;"
```

### 2. 浏览器准备
- 打开 Chrome/Edge 浏览器
- 访问 `http://localhost:1420/extensions`
- 打开开发者工具 (F12) → Network 标签，监控 API 请求

---

## 🧪 功能测试清单

### 1️⃣ 页面加载和布局

**测试步骤：**
1. 访问 `http://localhost:1420/extensions`
2. 观察页面加载过程

**验证点：**
- [ ] 页面标题显示"OpenClaw Desktop"
- [ ] 主标题"扩展商店"清晰可见
- [ ] 副标题"安装扩展插件，增强你的工作台功能"显示正确
- [ ] 统计徽章显示扩展总数和已安装数量
- [ ] 搜索框显示占位符文本"搜索扩展名称、描述或标签..."
- [ ] 分类筛选按钮显示：全部、开发工具、生产力、通讯、工具、文档
- [ ] Tab 切换显示：扩展商店、已安装
- [ ] 扩展卡片网格布局正确（响应式：1列/2列/3列）
- [ ] 无错误信息在控制台显示

**预期结果：**
- 页面在 2 秒内完全加载
- 所有 UI 元素正确渲染
- 无 JavaScript 错误

---

### 2️⃣ 搜索功能

**测试步骤：**
1. 在搜索框中输入"翻译"
2. 观察搜索结果
3. 清空搜索框
4. 输入不存在的关键词如"xyz_nonexistent_12345"

**验证点：**
- [ ] 输入"翻译"后，只显示包含"翻译"的扩展
- [ ] 搜索结果实时更新（无需点击搜索按钮）
- [ ] 第一个搜索结果包含"翻译"关键词
- [ ] 清空搜索后，恢复显示所有扩展
- [ ] 搜索不存在关键词时，显示空状态："未找到匹配的扩展"
- [ ] 空状态显示提示："尝试调整搜索条件或浏览其他分类"

**API 验证：**
- Network 标签中查看 `/api/extensions?search=翻译` 请求
- 响应状态码应为 200
- 返回数据应包含匹配的扩展

---

### 3️⃣ 分类筛选

**测试步骤：**
1. 点击"开发工具"分类按钮
2. 观察筛选结果
3. 依次点击其他分类按钮
4. 最后点击"全部"

**验证点：**
- [ ] 点击"开发工具"后，只显示该分类的扩展
- [ ] 选中的分类按钮高亮显示（蓝色背景）
- [ ] 其他分类按钮保持默认样式
- [ ] 点击"全部"后，显示所有扩展
- [ ] 分类筛选与搜索可以同时工作（组合过滤）

**UI 验证：**
- 选中按钮应有 `bg-primary` 类
- 未选中按钮应有 `bg-white` 类
- 按钮悬停时有缩放效果（`scale-105`）

---

### 4️⃣ Tab 切换

**测试步骤：**
1. 默认在"扩展商店" tab
2. 点击"已安装" tab
3. 观察内容变化
4. 再次点击"扩展商店" tab

**验证点：**
- [ ] 默认显示"扩展商店" tab，且高亮
- [ ] 点击"已安装"后，tab 高亮切换
- [ ] "已安装" tab 只显示 `installed=true` 的扩展
- [ ] 切回"扩展商店"后，显示所有可用扩展
- [ ] Tab 切换动画流畅（200ms 过渡）
- [ ] Tab 计数正确更新（如：扩展商店 (15)、已安装 (3)）

---

### 5️⃣ 扩展卡片显示

**测试步骤：**
1. 观察任意扩展卡片的完整信息

**验证点：**
- [ ] **图标**：圆形背景，渐变效果，悬停时放大
- [ ] **名称**：粗体显示，最多一行，溢出省略
- [ ] **作者**：显示 "by {author}"
- [ ] **描述**：最多两行，溢出显示省略号，有 title 属性显示完整文本
- [ ] **评分**：星级显示（★），数字评分
- [ ] **下载量**：带下载图标，格式化显示（如 1.5k, 2.3万）
- [ ] **版本号**：Badge 样式，等宽字体（如 v1.2.3）
- [ ] **标签**：最多显示 3 个，带 # 前缀，悬停变色
- [ ] **更新时间**：底部显示，格式为"更新于 2024年1月1日"
- [ ] **顶部装饰条**：悬停时显示渐变色条
- [ ] **已安装徽章**：右上角绿色徽章（仅已安装扩展）
- [ ] **圆角**：卡片整体圆角为 `rounded-2xl`（16px）

**交互验证：**
- [ ] 卡片悬停时向上移动（`hover:-translate-y-1`）
- [ ] 卡片悬停时阴影加深（`hover:shadow-xl`）
- [ ] 卡片悬停时名称颜色变为主题色

---

### 6️⃣ 安装扩展

**测试步骤：**
1. 找到一个未安装的扩展（无"已安装"徽章）
2. 点击"安装扩展"按钮
3. 观察按钮状态变化
4. 检查 Network 标签中的 API 请求

**验证点：**
- [ ] 按钮显示"安装扩展"文本和下载图标
- [ ] 按钮有渐变背景和阴影效果
- [ ] 点击后发送 POST 请求到 `/api/extensions/{id}/install`
- [ ] 请求成功后，按钮变为"禁用"/"启用"和"卸载"按钮
- [ ] 卡片右上角出现绿色"已安装"徽章
- [ ] "已安装" tab 中的计数增加 1

**API 验证：**
```json
POST /api/extensions/ext_translation/install
Response: { "success": true, "message": "扩展安装成功" }
```

---

### 7️⃣ 卸载扩展

**测试步骤：**
1. 找到一个已安装的扩展
2. 点击"卸载"按钮
3. 观察确认对话框
4. 点击"取消"（避免实际删除）

**验证点：**
- [ ] 点击卸载按钮后，弹出确认对话框
- [ ] 对话框消息包含："确定要卸载 "{extension.name}" 吗？"
- [ ] 对话框消息包含："此操作将移除该扩展及其所有配置数据。"
- [ ] 点击"取消"后，扩展保持安装状态
- [ ] 点击"确定"后，发送 DELETE 请求到 `/api/extensions/{id}/uninstall`
- [ ] 卸载成功后，按钮恢复为"安装扩展"
- [ ] "已安装"徽章消失
- [ ] "已安装" tab 中的计数减少 1

**API 验证：**
```json
DELETE /api/extensions/ext_translation/uninstall
Response: { "success": true, "message": "扩展卸载成功" }
```

---

### 8️⃣ 启用/禁用扩展

**测试步骤：**
1. 找到一个已安装且启用的扩展
2. 点击"禁用"按钮
3. 观察按钮状态变化
4. 再次点击"启用"按钮

**验证点：**
- [ ] 启用状态下，按钮显示"禁用"和电源图标
- [ ] 禁用状态下，按钮显示"启用"和电源图标
- [ ] 点击"禁用"后，发送 POST 请求到 `/api/extensions/{id}/disable`
- [ ] 禁用成功后，按钮变为"启用"
- [ ] 点击"启用"后，发送 POST 请求到 `/api/extensions/{id}/enable`
- [ ] 启用成功后，按钮变为"禁用"
- [ ] 按钮样式随状态变化（启用时为 outline，禁用时为 secondary）

**API 验证：**
```json
POST /api/extensions/ext_translation/disable
Response: { "success": true, "enabled": false }

POST /api/extensions/ext_translation/enable
Response: { "success": true, "enabled": true }
```

### 9️⃣ 配置弹窗

**测试步骤：**
1. 找到一个已安装的扩展
2. 点击设置图标按钮（齿轮图标）
3. 观察配置弹窗
4. 验证弹窗内容和样式
5. 点击"关闭"按钮

**验证点：**
- [ ] 点击设置按钮后，弹窗立即显示
- [ ] 弹窗背景有半透明黑色遮罩（`bg-black/50`）
- [ ] 弹窗背景有模糊效果（`backdrop-blur-sm`）
- [ ] 弹窗圆角为 `rounded-2xl`（16px）✅ **关键修复点**
- [ ] 弹窗最大宽度为 `max-w-md`（448px）
- [ ] 弹窗最大高度为 `max-h-[90vh]`，超出可滚动
- [ ] 弹窗不会溢出视口（使用 Portal 渲染到 body）

**弹窗内容验证：**
- [ ] **头部**：显示扩展图标、名称、"扩展配置"副标题
- [ ] **头部右侧**：有关闭按钮（X 图标）
- [ ] **基本信息区域**：
  - [ ] 版本号（v1.2.3）
  - [ ] 作者名称
  - [ ] 状态（已启用/已禁用，带颜色区分）
- [ ] **描述区域**：完整显示扩展描述
- [ ] **标签区域**：显示所有标签
- [ ] **主页链接**：如果有 homepage，显示"访问主页"链接
- [ ] **底部**：有"关闭"按钮

**交互验证：**
- [ ] 点击头部 X 按钮可以关闭弹窗
- [ ] 点击底部"关闭"按钮可以关闭弹窗
- [ ] 点击遮罩层外部可以关闭弹窗（可选）
- [ ] 弹窗打开时有淡入动画（`animate-in fade-in zoom-in`）
- [ ] 弹窗关闭时有淡出动画

**样式验证：**
- [ ] 弹窗 z-index 为 `z-[9999]`，确保在最上层
- [ ] 弹窗阴影为 `shadow-2xl`
- [ ] 头部有渐变背景（`from-gray-50 to-white`）
- [ ] 底部有灰色背景（`bg-gray-50`）
- [ ] 内容区域可滚动（`overflow-y-auto`）

## 📝 注意事项

1. **禁止 Mock 数据**：所有测试必须使用真实的后端 API 和数据库数据
2. **清理测试数据**：测试完成后，清理创建的测试数据
3. **网络监控**：始终打开 Network 标签监控 API 请求
4. **控制台检查**：检查 Console 标签是否有错误或警告
5. **多浏览器测试**：至少在 Chrome 和 Edge 上测试
6. **不同分辨率**：测试多种屏幕尺寸（手机、平板、桌面）
7. **慢速网络**：模拟 3G 网络测试加载状态
8. **无障碍测试**：使用 Lighthouse 检查可访问性
