# Context Monitor Hook

## 用途
监控对话上下文使用情况，当上下文即将耗尽时发出警告。

## 触发条件
- 对话轮次 > 15
- 上下文使用率 > 70%

## 警告阈值

| 级别 | 上下文使用率 | 行动 |
|------|-------------|------|
| INFO | 50-70% | 继续正常操作 |
| WARN | 70-85% | 建议开始compact |
| CRITICAL | > 85% | 必须compact |

## 检测规则

```javascript
// 当以下条件满足时触发警告：
1. 最近 5 条消息总 token 增长 > 10000
2. 当前文件变更涉及 > 3 个文件
3. Agent 调用链深度 > 3

// 建议的compact策略：
- 按文件类型分组compact（前端/后端/配置）
- 保留最新的 10 轮对话
- 归档中间结果到 .peaks/checkpoints/
```

## 自动行为

1. **轻度警告** (WARN):
   - 显示 "⚠️ 上下文使用率: XX%"
   - 建议用户可以继续或手动compact

2. **重度警告** (CRITICAL):
   - 显示 "🔴 上下文即将耗尽"
   - 提示将自动进行compact
   - 准备好调用 /compact 命令

## 输出格式

```
📊 Context Status:
- 使用率: [████░░░░░░] 45%
- 轮次: 12/50
- 预计剩余: ~20 轮
```
