## ⚠️ 部分通过/待修复的问题

### Issue 1: 禁用功能UI反馈不明显
**问题描述**:
- 点击"禁用"按钮后，按钮文本仍显示"禁用"
- 没有明显的视觉反馈表明扩展已被禁用
- 预期应该变为"启用"按钮

**当前行为**:
```
点击前: [禁用] [设置] [卸载]
点击后: [禁用(active)] [设置] [卸载]  // 只有active状态变化
```

**预期行为**:
```
点击前: [禁用] [设置] [卸载]
点击后: [启用] [设置] [卸载]  // 按钮文本应改变
```

**影响**: 用户无法直观判断扩展是否已禁用

**优先级**: 🔴 高（用户体验问题）

**建议修复**:
```typescript
// 在ExtensionCard组件中
{extension.enabled ? (
  <Button onClick={() => onDisable(extension.id)}>禁用</Button>
) : (
  <Button onClick={() => onEnable(extension.id)}>启用</Button>
)}
```

---

### Issue 2: 控制台错误（非阻塞）
**错误类型**:
1. Gateway ping方法不存在（健康检查失败）
2. Tauri API未定义（浏览器环境不支持）

**影响**: 不影响扩展商店功能，但会在控制台产生噪音

**建议**: 
- 在浏览器环境中禁用Tauri相关代码
- 实现Gateway的ping方法或移除健康检查
