# 专家系统使用指南

## 📋 概述

专家系统允许你创建和管理多个 AI 专家角色，每个专家都有独特的系统提示词（System Prompt），可以针对特定场景提供专业化的帮助。

---

## ✨ 功能特性

### 1. 创建自定义专家
- **名称**: 专家的名称（必填）
- **描述**: 简短说明专家的用途
- **系统提示词**: 定义专家的行为、角色和专业领域
- **图标**: Emoji 图标用于视觉识别
- **颜色**: 主题色用于区分不同专家

### 2. 管理专家
- ✅ **编辑专家**: 修改任何字段
- ✅ **删除专家**: 移除不再需要的专家（默认专家不可删除）
- ✅ **设为当前**: 激活某个专家作为当前会话的默认助手
- ✅ **查看列表**: 浏览所有已创建的专家

### 3. 预设专家模板
系统预置了 5 个专业专家：
- 🤖 **通用助手**: 全能型 AI 助手
- 💻 **代码审查专家**: 代码质量和最佳实践
- ✍️ **写作助手**: 文本优化和编辑
- 📊 **数据分析专家**: 数据解读和洞察
- 🔒 **安全顾问**: 网络安全和漏洞评估

## 🚀 快速开始

### 初始化默认专家

运行以下 SQL 脚本来添加预设专家：

```bash
psql -U postgres -d openclaw -f scripts/seed-experts.sql
```

或者在 Docker 环境中：

```bash
docker exec -i peaksclaw-postgres psql -U postgres -d openclaw < scripts/seed-experts.sql
```

### 创建第一个自定义专家

1. 点击 **"创建专家"** 按钮
2. 填写专家信息：
   ```
   名称: 前端开发专家
   描述: 专注于 React、TypeScript 和现代前端技术栈
   系统提示词: 
     你是一位资深前端开发工程师，精通：
     - React 生态（Hooks, Context, Redux）
     - TypeScript 类型系统
     - 性能优化和最佳实践
     - 可访问性（a11y）
     
     你应该：
     1. 提供生产级别的代码示例
     2. 解释技术选型的原因
     3. 指出潜在的性能问题
     4. 推荐合适的工具和库
   图标: ⚛️
   颜色: #61DAFB
   ```
3. 点击 **"创建"**

## 💡 使用场景

### 场景 1: 代码审查

**选择专家**: 💻 代码审查专家

**提问示例**:
```
请审查这段 React 组件代码，指出潜在问题：

const UserProfile = ({ userId }) => {
  const [user, setUser] = useState(null);
  
  useEffect(() => {
    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(setUser);
  }, []);
  
  return <div>{user?.name}</div>;
};
```

**预期回答**: 专家会指出缺少错误处理、loading 状态、依赖数组等问题。

---

### 场景 2: 技术写作

**选择专家**: ✍️ 写作助手

**提问示例**:
```
帮我改进这段技术文档的描述，让它更清晰易懂：

"This function takes a parameter and does stuff with it to make things work better."
```

**预期回答**: 提供更专业、清晰的表述，并解释改进原因。

---

### 场景 3: 数据分析

**选择专家**: 📊 数据分析专家

**提问示例**:
```
我有以下销售数据，请帮我分析趋势：

月份 | 销售额
1月  | 100万
2月  | 120万
3月  | 95万
4月  | 150万

应该关注哪些指标？如何可视化？
```

**预期回答**: 提供统计分析、趋势解读和可视化建议。

## 🔧 高级技巧

### 1. 设计有效的系统提示词

**好的提示词结构**:
```markdown
你是一位[角色]，专注于[领域]。

你的职责包括：
1. [职责1]
2. [职责2]
3. [职责3]

你应该：
- [行为准则1]
- [行为准则2]
- [行为准则3]

重点关注：
- [重点1]
- [重点2]
```

**示例**:
```
你是一位 UX/UI 设计专家，专注于用户体验设计和界面优化。

你的职责包括：
1. 评估界面的可用性和可访问性
2. 提供设计改进建议
3. 解释设计决策的用户心理学依据

你应该：
- 优先考虑用户需求
- 平衡美观和功能
- 提供具体的改进方案
- 引用设计原则和最佳实践

重点关注：
- 视觉层次和信息架构
- 交互反馈和微动画
- 色彩理论和对比度
- 响应式设计
```

### 2. 专家切换策略

- **日常任务**: 使用通用助手
- **专业工作**: 切换到对应领域的专家
- **复杂项目**: 组合多个专家的建议

### 3. 组织专家库

**建议的分类**:
- 🎯 **核心专家**: 3-5 个常用专家
- 🔧 **专业工具**: 针对特定任务的专家
- 🧪 **实验性**: 测试新的提示词配置

## ❓ 常见问题

### Q: 如何删除专家？
A: 点击专家卡片上的垃圾桶图标。**注意**: 默认专家无法删除。

### Q: 可以导出/导入专家吗？
A: 目前不支持，计划在未来版本中添加。

### Q: 专家会影响聊天历史吗？
A: 不会。切换专家只会改变 system prompt，不会影响已有消息。

### Q: 如何测试专家效果？
A: 创建专家后，立即在聊天中测试，观察 AI 的回答是否符合预期。如果不满意，可以编辑系统提示词进行调整。

### Q: 最多可以创建多少个专家？
A: 目前没有硬性限制，但建议保持在 20 个以内以便管理。

## 📝 最佳实践

### ✅ 推荐做法

1. **明确角色定位**: 每个专家应该有清晰的职责范围
2. **具体化行为准则**: 告诉 AI 应该如何思考和回应
3. **迭代优化**: 根据实际使用效果调整提示词
4. **保持一致性**: 相似的专家使用相似的结构
5. **添加示例**: 在提示词中包含典型问答示例

### ❌ 避免的做法

1. **过于宽泛**: "你是一个聪明的助手" 太模糊
2. **矛盾指令**: 不要同时要求"简洁"和"详细"
3. **过度约束**: 给 AI 太多限制会降低灵活性
4. **忽略测试**: 创建后一定要实际测试效果
5. **忘记更新**: 随着需求变化，及时更新专家配置
