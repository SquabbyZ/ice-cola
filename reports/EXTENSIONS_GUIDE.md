# 扩展商店使用指南

## 📋 概述

扩展商店允许你浏览、安装和管理各种功能扩展插件，以增强 OpenClaw 工作台的能力。每个扩展都提供特定的集成功能，如 GitHub、Notion、Slack 等第三方服务的连接。

---

## ✨ 功能特性

### 1. 浏览扩展
- **分类浏览**: 按类别筛选（开发工具、生产力、通讯、工具、文档）
- **搜索功能**: 通过名称、描述或标签搜索扩展
- **详细信息**: 查看评分、下载量、版本、作者等信息

### 2. 安装管理
- ✅ **一键安装**: 点击安装按钮即可添加扩展
- ✅ **启用/禁用**: 临时停用扩展而不卸载
- ✅ **卸载扩展**: 完全移除不需要的扩展
- ✅ **配置设置**: 访问扩展的配置页面（待实现）

### 3. 已安装管理
- 查看所有已安装的扩展
- 快速启用/禁用
- 访问扩展设置
- 查看更新状态

---

## 🚀 快速开始

### 浏览扩展

1. 导航到 **"扩展商店"** 页面
2. 使用顶部的**搜索框**查找特定扩展
3. 点击**分类标签**筛选感兴趣的类型
4. 浏览扩展卡片查看详细信息

### 安装扩展

1. 找到想要安装的扩展
2. 点击 **"安装"** 按钮
3. 等待安装完成（显示"已安装"徽章）
4. 扩展自动启用，立即可用

### 管理已安装扩展

切换到 **"已安装"** 标签页：

- **禁用扩展**: 点击"禁用"按钮临时停用
- **启用扩展**: 点击"启用"按钮重新激活
- **卸载扩展**: 点击"卸载"按钮完全移除
- **配置扩展**: 点击设置图标打开配置（待实现）

## ❓ 常见问题

### Q: 如何知道扩展是否安全？
A: 
- 查看评分和下载量（高评分、多下载更可靠）
- 检查作者信息（官方扩展更可信）
- 阅读扩展描述和功能说明
- 查看最后更新时间（近期更新更活跃）

### Q: 安装扩展会影响性能吗？
A: 
- 禁用的扩展不会消耗资源
- 建议只启用正在使用的扩展
- 定期清理不用的扩展

### Q: 扩展会自动更新吗？
A: 
- 当前版本需要手动更新（未来版本会支持自动更新）
- 在"已安装"标签页查看可用更新
- 点击扩展卡片查看最新版本

### Q: 可以自定义扩展配置吗？
A: 
- 当前版本配置功能正在开发中
- 未来将支持详细的自定义设置
- 部分扩展可能提供基础配置选项

### Q: 如何报告扩展问题？
A: 
1. 检查扩展主页或仓库链接
2. 提交 Issue 描述问题
3. 提供错误信息和复现步骤
4. 联系扩展作者或社区

### Q: 可以开发自己的扩展吗？
A: 
- 是的！我们欢迎社区贡献
- 查看开发者文档了解扩展开发指南
- 提交 PR 将你的扩展加入商店

## 📝 最佳实践

### ✅ 推荐做法

1. **定期审查**: 每月检查已安装扩展，卸载不用的
2. **阅读描述**: 安装前仔细阅读功能和权限说明
3. **测试扩展**: 新安装的扩展先测试基本功能
4. **备份配置**: 重要扩展的配置定期备份
5. **关注更新**: 及时更新扩展以获得新功能和安全修复

### ❌ 避免的做法

1. **盲目安装**: 不要安装不明来源的扩展
2. **过度安装**: 保持扩展数量合理，避免冗余
3. **忽略权限**: 注意扩展请求的权限范围
4. **忘记禁用**: 不用的扩展及时禁用或卸载
5. **跳过阅读**: 不要跳过更新日志和变更说明

## 🛠️ 开发者指南

### 创建扩展

扩展的基本结构：

```typescript
interface Extension {
  id: string;              // 唯一标识符
  name: string;            // 扩展名称
  description: string;     // 简短描述
  version: string;         // 版本号
  author: string;          // 作者
  category: string;        // 分类
  icon: string;            // 图标 (emoji)
  rating: number;          // 评分 (0-5)
  downloads: number;       // 下载量
  tags: string[];          // 标签
  homepage?: string;       // 主页链接
}
```

### 提交扩展

1. Fork 扩展仓库
2. 按照模板创建扩展
3. 编写文档和测试
4. 提交 Pull Request
5. 等待审核和合并
